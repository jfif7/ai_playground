description: Phase 2 - System Design & Use Cases
---

> **Note:** Consult the `document_management` skill for file naming and organization rules.

1. [ ] **Analyze Requirements**
   - Read `docs/requirements.md`.

2. [ ] **Draft System Design**
   - Create/Update `docs/design.md`.
   - Define: Data Models, API Signatures, Flowcharts (Mermaid).

3. [ ] **Breakdown Use Cases**
   - Create individual spec files in `docs/specs/`.
   - Example: `docs/specs/uc_01_user_login.md`.
   - Each spec must contain: Pre-conditions, Steps, Post-conditions.

4. [ ] **User Review**
   - Ask user to review the design and the breakdown.
   - **STOP** and wait for approval.
